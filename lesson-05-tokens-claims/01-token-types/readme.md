## ğŸ¯ Exam Tips

> **Remember:**
> - All B2C tokens are **JWTs** (JSON Web Tokens)
> - **ID Token** = Authentication (who is the user)
> - **Access Token** = Authorization (what can they do)
> - **Refresh Token** = Get new tokens silently
> - Tokens must be **validated** by checking signature, issuer, audience, expiration
> - Access tokens contain **scopes** (`scp` claim)
> - The `sub` claim is the user's **object ID**
